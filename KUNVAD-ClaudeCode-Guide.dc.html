Claude Code × GitHub × Vercel

Development & Deployment Workflow — KUNVAD Project

Overview

This guide walks you from downloading the current project files all the way to a live auto-deploy on Vercel, with Claude Code handling file edits via natural language in the terminal.

Stack: Static HTML/DC files · GitHub (source of truth) · Vercel (hosting) · Claude Code CLI (AI editing)

Phase 1 — One-Time Setup

1
Download the project Export the project as a ZIP from this tool, unzip it into a local folder named kunvad.
2
Initialize Git & push to GitHub 
cd kunvad
git init
git add .
git commit -m "initial commit"
gh repo create kunvad --public --source=. --push
# or: git remote add origin https://github.com/YOU/kunvad.git && git push -u origin main
3
Connect Vercel to the repo
  1. Go to vercel.com/new → Import Git repository → select kunvad
  2. Framework: Other (static files, no build step)
  3. Output directory: leave blank (./)
  4. Click Deploy — Vercel will auto-deploy on every push to main
4
Install Claude Code CLI 
npm install -g @anthropic-ai/claude-code
Requires Node 18+. Authenticate once: 
claude auth login
5
Write a CLAUDE.md context file Place CLAUDE.md in the project root — Claude Code reads it automatically at the start of every session: 
# KUNVAD Project

## Files
- KUNVAD-Home.dc.html     — Home page (hero, evasive buttons)
- KUNVAD-Me.dc.html       — About page
- KUNVAD-Blog.dc.html     — Blog page
- KUNVAD-Project.dc.html  — Project page
- KUNVAD-Hi.dc.html       — Hi/Contact page
- KUNVAD-CMS.dc.html      — Content management panel

## Content
All editable content is stored in browser localStorage under the key
`kunvad_cms`. Static defaults live in each page's `_D` object.

## Deploy
git push origin main → Vercel auto-deploys.

Phase 2 — Daily Edit Workflow

6
Start a Claude Code session 
cd kunvad
claude
7
Prompt Claude Code to edit files Natural language works. Examples: 
# Change blog page heading
"In KUNVAD-Blog.dc.html, change the default blogHeading
 in _D from IT'S BLOG to STORIES."

# Add a new nav link
"Add a 'Contact' link to the nav in all inner pages
 pointing to mailto:tap@kunvad.com"

# Update CMS default bio text
"In KUNVAD-Me.dc.html update the default body text
 in _D.me to: [your new bio here]"
Claude Code will read the file, make the targeted change, and show a diff for approval.
8
Review the diff, then commit & push 
git add .
git commit -m "update blog heading + me bio"
git push origin main
Vercel picks up the push and deploys in ~30 seconds.

Phase 3 — Useful Prompt Patterns

Edit default content (visible before CMS override)

"In KUNVAD-Hi.dc.html, change _D.hi.body to: [new text]"

Change theme defaults

"In all KUNVAD-*.dc.html files, change the default
 accent color in _D.theme from rgb(245,192,24) to rgb(255,180,0)"

Add a new page

"Create KUNVAD-Work.dc.html following the same pattern
 as KUNVAD-Blog.dc.html, with heading WORK and
 status 'Coming soon'. Add it to the nav in all pages."

Deploy with a message

"After editing, run: git add . && git commit -m 'add work page' && git push"
Tip: You can ask Claude Code to do the git commit and push in the same prompt — e.g. "Make the change and push to GitHub." It will stage, commit, and push automatically.
